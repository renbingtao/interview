
一般有3个工程：消费者，提供者，api接口。消费者和提供者都接入api接口


dubbo集成springboot
提供者示例
启动类添加@EnableDubbo
@DubboService
public class DemoServiceImpl implements DemoService {
    @Override
    public String sayHello(String name) { return "Hello " + name; }
}
配置文件
dubbo:
  application:
      name: dubbo-springboot-demo-provider
      logger: slf4j
  protocol:
    name: tri
    port: 50052
  registry:
    address: nacos://${nacos.address:127.0.0.1}:8848?username=nacos&password=nacos


消费者示例
启动类添加@EnableDubbo
@DubboReference
private DemoService demoService;
配置文件
dubbo:
  application:
      name: dubbo-springboot-demo-consumer
      logger: slf4j
      qos-port: 33333
  registry:
    address: nacos://${nacos.address:127.0.0.1}:8848?username=nacos&password=nacos


dubbo.application:每个应用的application.name应不同
dubbo.registry:用来定义服务注册中心
dubbo.protocol:用来表示通信协议
