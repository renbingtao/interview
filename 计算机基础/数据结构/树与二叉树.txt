
==============树的定义==============

树是n(n>=0)个结点的有限集。n=0时，称为空树。任意一颗非空树，必须满足：
1.有且仅有一个特定的结点，称为【根结点】
2.n>1时，其余结点可分为m个互不相交的有限集，其中每个集合本身又是一颗树，称为根的【子树】

树的定义是递归的。树是一种典型的递归数据结构
树的特征：
1.根结点没有前驱，根结点以外的每个结点有且仅有一个前驱
2.所有结点均可拥有零个或多个后继

树适用于表示具有层次结构的数据
在包含n个结点的树中，恰好存在n-1条边

==============基本概念==============

【祖先】根结点A到结点K的唯一路径上经过的所有其他结点。相对应的为【子孙】。一个结点可以有多个祖先和多个子孙
【双亲】路径上最近的结点。反之称为【孩子】。根结点是唯一没有双亲的结点
【兄弟】具有相同双亲的结点
【堂兄弟】位于同一层的结点们，互为堂兄弟
【层次】根为1层，其孩子为2层。结点的深度即为其所在的层次
【树的高度】树中结点的最大层数
【结点的高度】以某个结点为根结点的树的高度
【结点的度】树中一个结点的【孩子】个数。注意是孩子而非子孙，即二叉树的某结点的度为0，1，2
【树的度】树中所有结点的度的最大值
【分支结点】也叫非终端结点/叶子结点，度>0的结点。每个结点的分支数即为度
【叶结点】也叫终端结点，度=0的结点
【有序树/无序树】树中各个结点的子树从左到右具有固定次序，不可互换，则为有序树，否则无序树。对于一个有序树，若交换某结点的子结点位置，则得到一颗不同的树
【路径/路径长度】从一个结点到另一个结点所经过的结点序列，即为路径。该路径上【边】的数目则为路径长度
【树的路径长度】树根到每个结点的路径长度的总和
【森林】森林是m棵互不相交的树的集合。将根节点移除，其各子树构成一个森林；反之，为m棵独立的树添加一个新的结点并作为这m棵树的共同根，则森林转化为一棵树

总结点数-1=总分支数=度数总和

==============树的性质==============

树的结点数n = 所有结点的度数之和 + 1
理解：所有结点的度数之和=边数。再加上根结点，即为结点数

度为m的树中，第i层上至多有 m的(i-1)次方 个结点

高度为h的m叉树至多有 (m的h次方-1)/(m-1)个结点
即1+m+m的平方+m的三次方+....+m的h-1次方=(m的h次方-1)/(m-1)

度为m，结点为n的树的最大/最小高度
最小高度：树尽可能饱满，所有结点的度都是m
最大高度：树只有一个结点的度为m，其他结点的度都是1

度数      数量
0       N0
1       N1
2       N2
3       N3
....
m       Nm
总度数N-1=1*N1+2*N2+3*N3+....+m*Nm
总结点数N=N0+N1+...+Nm
相减可得：N0=N2+2*N3+3*N4+....+(m-1)*Nm+1
上述即为叶子结点的总数
可知：叶子数量和度为 1 的结点无关，只由度≥2 的结点贡献
举例：一棵树有 3 个度 2 结点、2 个度 3 结点，则叶子结点数量为
n0=1+1*3+2*2=8

==============二叉树==============

定义：每个结点最多有两颗子树（度最多为2），且子树有左右之分，次序固定，不可交换
二叉树属于有序树，即使结点只有一颗子树，也要区分是左子树还是右子树

二叉树 和 度=2的有序树 有本质区别
1.度=2的树至少有3个结点，而二叉树可以为空
2.度=2的树，若某个结点只有一个孩子，则【无须】区分左右次序；而在二叉树中，即使只有一个孩子也要区分左右子树

【满二叉树】
每一层都包含了该层能容纳的最大结点数。若高度为h，则结点数为2的h次方-1
对满二叉树进行编号：
            1
    2               3
4       5       6       7
对于编号为i的结点，双亲编号为i/2（向下取整）；左孩子编号为2i；右孩子编号为2i+1

【完全二叉树】
和相同高度的满二叉树相比，每个结点一一对应
直观图像：
除最后一层外，前面所有层都是满的
最后一层节点靠左紧密排列，不能有空缺
比如
            1
    2               3
4       5

【二叉排序树】
左子树上所有结点的关键字均小于根结点的关键字
右子树上所有结点的关键字均大于根结点的关键字
左右子树本身也是二叉排序数

【平衡二叉树】
树中任意一个结点的左子树和右子树的高度之差的绝对值不超过1

【正则二叉树】
树中每个分支结点（叶子结点）都有2个孩子，即结点的度只能为0或2

==============二叉树的性质==============

【非空二叉树】
叶子结点数=度为2的结点数+1，即n0=n2+1
第k层最多有2的k-1次方个结点。可以拓展：m叉树的第k层最多有m的k-1次方个结点，高度为h的m叉树最多有(mh -1)/(m-1)个结点

【完全二叉树】
按从上到下，从左到右编号 1，2，....，n，有以下关系
【最后一个分支结点】的编号为n/2（向下取整）。小于等于该值的是分支结点，否则是叶子结点
叶子结点只可能出现在最后两层上
若存在度为1的结点，则最多只可能有一个，且只能为【最后一个分支结点】（n/2向下取整），且有左孩子无右孩子
某个结点i为叶结点或仅有左孩子，则大于该结点编号的结点均为叶子结点
若n为奇数，则所有结点都有左右孩子；若为偶数，则【最后一个分支结点】仅有左结点
i结点的双亲为i/2向下取整，左孩子为2i，右孩子为2i+1
结点所在高度为(log2 i向下取整)+1
高度为h的完全二叉树结点总数范围：[2的h-1次方 - 2的h次方 -1]，闭区间

==============二叉树的存储结构==============

1.顺序存储结构
使用一维数组，将编号i的结点保存在下标为i-1的位置
特别适合完满二叉树和完全二叉树，既能节省空间，又能快速定位（包括父子、兄弟）。如下图
        A
      /   \
     B     C
    / \
   D   E
用数组表示为[A, B, C, D, E]

对于一般的二叉树，则需要插入若干 空结点，使其结构与同高度的完全二叉树一致
这种做法在最坏情况下效率极低，比如高度为h，仅有h个结点的单支树，却要占用2的h次方-1个存储单元。如下图，用0表示空结点
                1
              /   \
             2     3
              \     \
               4     5
              /
             6
[1,2,3,0,4,0,5,0,0,6]
【建议从数组下标1开始存储结点，保证数组下标与结点编号一致】

2.链式存储结构
由于顺序存储的空间利用率较低，二叉树通常采用链式存储结构，即用链表结点来存储二叉树中的每个结点
二叉链表至少包含3个域：左指针域lchild、数据域data、右指针域rchild
typedef struct BiTNode{
    ElemType data;
    struct BiTNode *lchild,*rchild;
}

在包含n个结点的二叉链表中，有2n个指针域，其中n-1个是非空的（除了父节点，每个结点都有一个父节点的指针指向它，所以是n-1）
因此空链域（空的指针域）数量为n+1
删除某个二叉链表的所有结点，最适合的方式是后序遍历，因为先处理左右子树，再处理根节点

==============二叉树的遍历和线索二叉树==============

遍历一颗二叉树的关键在于确定对根节点(N)、左子树(L)、右子树(R)的访问顺序
在遵循“先左后右”的原则下，常见的遍历方式有：先序遍历、中序遍历、后序遍历。这里的“序”是根结点在遍历序列中的位置

【先序遍历】
即对节点的处理是在它的诸多儿子节点被处理之前进行的。即【根左右】

【中序遍历】
先处理左子树，然后当前节点，最后处理右子树【左根右】

【后序遍历】
即先处理儿子节点，再处理本节点。即【左右根】

上述三种，无论采用哪种方式，每个结点都被访问且仅一次，时间复杂度为O(n)。在递归实现中，系统工作栈的深度=树的高度，在最坏情况下，空间复杂度为O(n)

上面的三种方式，按照定义来遍历还是太复杂了。有更直观的方式
                       *A
                B               C
            D       E       F       G
遍历时，按照A - B - D - E - A - C - E - G - A的顺序
从A的左边出发。如果是先序，则在其他节点的左边加标记，访问时，遇到标记，则输出
如果是中序，则在下面标记；如果是后序，则在右边标记。但都是从A的左边出发，不代表第一个输出A

【层次遍历】
入下图，从上到下，从左到右逐层访问，顺序为ABCDE
1        A
       /   \
2     B     C
     / \
3   D   E
层次遍历需要借助一个队列，基本思想为：根结点入队，循环：【对头出队，访问值，左孩子入队，右孩子入队】
void LevelOrder(BiTree T){
    InitQueue(Q);                   //初始化辅助队列
    BiTree p;
    EnQueue(Q);                     //根结点入队
    while(!IsEmpty(Q)){
        DeQueue(Q,p);               //队头结点出队
        visit(p);                   //访问出队结点
        if(p->lchild!=NULL){
            EnQueue(p->lchild);     //左孩子不为空，则左孩子入队
        }
        if(p->rchild!=NULL){
            EnQueue(p->rchild);     //右孩子不为空，则右孩子入队
        }
    }
}

对于一颗给定的二叉树，上述4种遍历结果是唯一的。然而，仅凭4种的任意一种，无法唯一地确定二叉树
【中序序列】+其他三种中的一种，则可以唯一确定二叉树
完全二叉树+4中遍历中的任意一种，都能唯一确定二叉树

深刻理解中序的图像：从左到右扫描
先序的图像：将每个结点的右子树都低于左子树，从上到下扫描
后序的图像：将每个结点的左子树都低于右子树，从下到上扫描

【先序+中序】
中序横着直接抄，先序竖着直接抄，并一一对应
假设先序为ABCDEF，中序为CBAEDF，则构图如下：
A           A
B       B
C   C
D                   D
E               E
F                       F
    C   B   A   E   D   F
将节点相连即为二叉树

【后序+中序】
后序是反着写
假设后序为DABEC，中序为DEBAC，则构图如下：
C                   C
E       E
B           B
A               A
D   D
    D   E   B   A   C

【层序+中序】
根据层序的顺序，依次入座
假设层序为ABCDE，中序为DBEAC，则构图如下：
                A
        B           C
    D       E
    D   B   E   A   C

==============线索二叉树==============

二叉树的遍历：按照特定规则将树中所有结点排列成一个线性序列。从而得到相应的遍历序列。在该序列中，除了首尾结点，每个结点斗鱼唯一的直接前驱和直接后继

线索二叉树的结点结构
lchild      ltag    data    rtag    rchild

若某结点无左子树，则将lchild指向前驱结点
若某结点无右子树，则将rchild指向后继结点

ltag=0表示lchild指向左孩子，ltag=1表示lchild指向前驱
ltag=1表示rchild指向右孩子，ltag=1表示lchild指向后继
最左边的结点的左指针为空，且ltag=1（虽然左孩子也是空，但ltag要记为0），表示它的前驱为空；最右边的结点的右指针为空，且rtag=1，表示它的后继为空

采用上述结构的二叉链表称为【线索链表】，指向前驱或后继的指针称为【线索】。这种带有线索的二叉树称为【线索二叉树】

二叉树的线索化
将二叉链表中的空指针域替换为指向前驱或后继的线索

【中序线索二叉树】
中序线索二叉树的构造
指针pre指向刚刚访问过的结点，指针p指向当前正在访问的结点，pre为p的中序前驱。中序遍历过程中，若p的左指针为空，则令左指针=pre；若pre的右指针为空，则右指针=p
为了方便遍历，可在线索链表上增设一个【头结点】，其lchild指向根结点；rchild指向中序序列的最后一个结点；第一个节点的lchild指向头结点；最后一个结点的rchild指向头结点
如此一来，便形成一个双向循环线索链表，支持正向和反向的高效遍历
中序二叉树的遍历
1.找到第一个节点
即一直向左遍历，直到ltag=1
ThreadNode *Firstnode(ThreadNode *p){
    while(p->ltag==0){
        p=p->lchild;
    }
    return p;
}
2.从第一个节点开始，依次查找后继，直到后继为空
ThreadNode *Nextnode(ThreadNode *p){
    if(p->rtag==0){                     //是右孩子时，找到右子树中最左下结点
        return Firstnode(p->rchild);
    }
    return p->rchild;                   //rtag=1时，直接返回后继线索
}
总之，中序找前驱或者后继都是比较简单的

【先序线索二叉树】
构造过程和中序类似，只不过序列的排列，从中序改为了前序
查找后继：若该结点有左孩子，则左孩子为后继；若无左孩子但有右孩子，则右孩子为后继，若为叶子结点，则rchild直接指向后继
总之，先序找后继是比较简单的，找前驱比较麻烦

【后序线索二叉树】
构造过程和中序类似，只不过序列的排列，从中序改为了后序
通常采用带父指针的【三叉链表】作为存储结构，而非标准的二叉线索链表
后序找后继是很复杂的
