
========================图论算法========================

定义
【图G】由【顶点集V】和【边集E】组成，记G=(V,E)。其中V(G)表示G中顶点的有限非空集；E(G)表示G中边的集合
用|V|表示G中顶点的数量，|E|表示G中边的数量
【注意】线性表可以为空，树可以为空，但图不能为空图，即至少有一个顶点（可以没有边）

【有向图】
若E是有向边（也称弧），则图G为有向图。弧是顶点的有序对，记为<v,w>，其中v称为弧尾，w称为弧头，表示从v到w的有向边，也称v邻接到w
1   ->  2   ->  3
    <-
上面的有向图可表示为  G=(V1,E1)   V1={1,2,3}  E1={<1,2>,<2,1>,<2,3>}

【无向图】
若E是无向边（简称边）的有限集合，则G称为无向图。边是顶点的无序对，记为(v,w)或(w,v)。可以说w和v互为邻接点，也可以说边(v,w)和v,w相关联

【简单图】满足如下条件
1.不存在重复边。举例：有向图中，e1=<1,2>,e2=<1,2>，则e1和e2重复
2.不存在顶点到自身的边
仅讨论简单图

【顶点的度】
【无向图】顶点v的度是指依附于顶点v的边的数量，记为TD(v)。无向图的【全部顶点】的度之和=边数*2 （因为每个边和两个顶点相连）
【有向图】顶点v的度分为【入度】和【出度】。入度=以v为终点的有向边的数量，记为ID(v)，出度=以V为起点的有向边的数量，记为OD(v)
顶点V的度=入度+出度。有向图的全部顶点的入度之和=出度之和=边数，因为每条边都有一个起点和一个终点

【路径长度】
顶点a到b之间的一条路径指 顶点序列a,v1,v2,....,vm,b。路径所含【边的数量】称为路径长度

【回路】也叫环，指第一个顶点和最后一个顶点相同的路径
若无向图有n个顶点，且边的数量>=n --> 则此图一定有环
各个顶点的度>=2 --> 则此图一定有环

【简单路径】
路径序列中顶点不重复出现的路径

【简单回路】
除了第一个和最后一个顶点，其他顶点不重复的回路

【距离】
从顶点u到v的最短路径的长度。若路径不存在，则距离为无穷大

【子图】
有两个图G=(V,E)和G`=(V`,E`)，若V`是V的子集，E`是E的子集，且【E`的端点必须在V`中】，则称G`是G的子图
注意：并非V和E的任意子集都能构成G的子图

【生成子图】
V(G`)=V(G)的子图G`，即【顶点一个都不少】，则称其为G的生成子图

【连通】
在无向图中，若从顶点v到w存在路径，则称v和w是连通的
若G中任意两个顶点都是连通的（不一定是直接相连），则称G为【连通图】，否则为【非连通图】

【连通分量】
无向图中的极大连通子图。简单理解：非连通图拆成几块，就是几个连通分量。一个图有3个连通分量，可类比成一个森林里有3棵树
若一个无向图有n个顶点，若边的数量<=n-2，则图必然是【非连通的】
连通图的连通分量是它自己，数量一定是1

【强连通】
在有向图中，如果一对顶点v和w，从v到w和从w到v之间都有路径，则称这两个顶点是【强连通的】
图中任意一对顶点都是强连通的（不一定是直接相连），则称图为强连通图

【强连通分量】
有向图中的极大强连通子图
强连通图的强连通分量是它自己，数量一定是1

注意：在无向图中讨论连通性，在有向图中讨论强连通性

【生成树】
生成树是针对连通图而言的，是一个包含所有顶点的【极小连通子图】。所谓极小连通子图，可以理解为在顶点确定的前提下，包含了最少的边（即任意一个边都不能去掉，去掉就变成非连通）
若图中有n个顶点，则生成树一定包含n-1条边
对于生成树而言，移除任意一条边，都会变成非连通图；添加任何一条边都会形成一个回路

【生成森林】
针对非连通图而言的，每个连通分量的生成树共同构成了该图的生成森林

区分极大连通子图和极小连通子图
极大连通子图要求连通且顶点数及边数尽可能地多
极小连通子图要求连通且边数尽可能地少

【权值】
在一些图中，每条边可以被赋予一个代表某种意义的数值，称为该边的权值

【带权图】带有权值的图，也叫【网】

【带权路径长度】
路径上所有边的权值之和

【完全图】
对于无向图，顶点的数量为n，则边的数量为[0,n(n-1)/2]
当边的数量达到最大值时，该无向图称为【完全图】，意味着任意两个顶点之间都有一条边相连

【有向完全图】
对于有向图，顶点的数量为n，则边的数量为[0,n(n-1)]
当边的数量达到最大值时，该有向图称为【有向完全图】，意味着任意两个顶点之间都有两条方向相反的边相连

【稀疏图/稠密图】
顶点一定的前提下，边数较少的图称为【稀疏图】，反之为【稠密图】

【有向树】
一个顶点的入度=0，其他顶点的入度都为1的有向图

========================图的存储及基本操作========================









【邻接矩阵】
对于每条边（u，v），连通则A[u][v]=true，否则为false；如果有权则A[u][v]表示权。邻接矩阵的问题是空间复杂度O（n平方），如果边不是很多（稀疏的图），则浪费空间，因此适合稠密的图，即点少边多
例：无向图有3个顶点，则用一个一维数组表示顶点[a,b,c]；用一个二维数组表示顶点之间的关系，对角线（aa->cc）永远为0，关于对角线对称（有向图则不是）。比如ac有边，bc有边，则如下
    a   b   c
a   0   0   1
b   0   0   1
c   1   1   0
对于有向图，A[u][v]=1表示u指向v
无向图某顶点的入度/出度=该行或列 标志=1的个数。比如c的入度为2

下图为有向图
    a   b   c
a   0   0   1
b   1   1   0
c   1   0   0
有向图某顶点的入度=第v列的和，比如a的入度为2，bc的入度为1；出度=第u行的和，比如ac的出度=1，b的出度=2

【邻接表】
使用数组+链表，数组保存了全部的顶点，链表里保存了数组元素邻接的顶点。邻接表是表示图的标准方法
例：0->1  0->2    1->3    2->3        则邻接表形式如下
数组下标 + 链表
0   ->  node[1] ->  node[2]
1   ->  node[3]
2   ->  node[3]
3

【拓扑排序】
只适用于【有向无环图】，即有环图不存在拓扑排序
有拓扑排序 等价于 有向无环
对有向无圈图的顶点的一种排序，且先完成依赖，再做后面的事（即若存在边u->v，则排序中u一定在v前面）。拓扑排序的本质是将 DAG 的 “依赖关系” 转化为线性顺序，核心价值是解决 “有前置条件的任务调度” 问题
拓扑序不一定唯一
唯一举例：A->B->C
不唯一举例：A->C,B->C 则拓扑排序可以为：A,B,C 也可以为B,A,C

最短路径算法
即从A点到其他点的最短路径
解决单源最短路径问题的一般方法叫Dijkstra算法，是贪心算法，每次选择距离A最近的点。该算法每一次遍历都得到距离A最近的点，而不是针对某个点计算全部路径取最小值
思路：遍历A，找到距离最短的点B，标记为已解决（其他节点为CD）。接下来循环：遍历B（得到CEF），C=min(ABC,AC)，A到CDEF中最小的（比如是E）标记为已解决。再遍历E

网络流问题
顶点A到顶点B，从顶点A输入水流x，在顶点B全部流出，每个边有的权重为能够接收的最大水流（容量）。最大流问题是确定从A到B能够通过的最大流量
一种简单的最大流算法
找到任意A->B的路径，计算路径的最小权重（类比成木桶的最短板），该路径所有边减去权重，减为0时删除该边，差值作为容量。重复上述过程，直到不存在A->B的路径为止，此时的权重为空闲量。流量=容量-空闲量，再计算A的流出的流量总和，即为最大流
简单最大流算法的问题是，不能保证结果一定是最大的，结果和路径的先后顺序有很大关系
某个路径集合导致流量无法增大，则称为阻塞流。最大流也是一种阻塞流
Ford-Fulkerson算法可以保证一定取得最大流。在删除权重时，添加一个反方向的流，可以用来搜索A->B的路径，除此之外一样
Edmonds-Karp算法是对Ford-Fulkerson算法的一种特例，唯一改动是搜索A->B的路径时总是搜索最短路径（按照无权重图处理，即最小边数）

最小生成树
在无向图中找出一颗最小生成树（包含全部顶点），每个边都有权重，这里的最小指的是边的权重的和最小。一个树有n个点，n-1个边。最小生成树不唯一，但权重和是唯一的
prim算法
加点法，一开始只有一个顶点，在每一步都把一个节点当成根节点，将邻接的权重最小的节点加入树。点多时，要遍历的点就多，所以效率低，因此更适合稠密图
kruskal算法
加边法，一开始所有点都是孤立的，每次遍历所有的边（当前边和现有的树不能在一个连通分量里。举例：A-B-C-D在一个树里，则BD这个边和树是在一个连通分量里，因此不考虑），找到权重最小的边，连接起来。边多时效率低，因此更适合稀疏图

图的基础遍历方式
图的遍历核心是从指定起点出发，按规则访问图中所有顶点且不重复
主流方式有两种：深度优先搜索（BFS） 和 广度优先搜索（DFS）
【BFS】
可求出【无权图】中每一对顶点间的最短路径
求【带权图】最短路径必须使用松弛 (relax) 算法
